># 2. Быстрый старт

В этом разделе мы рассмотрим, как запустить приложение локально с помощью Docker. Это самый простой и рекомендуемый способ для большинства пользователей.

## Требования

- **Docker** и **Docker Compose**: [Инструкция по установке](https://docs.docker.com/get-docker/)
- **Git**: для клонирования репозитория
- **API-ключ** от LLM-провайдера (например, Google AI Studio для Gemini или OpenAI)

## Установка и запуск

1.  **Клонируйте репозиторий:**

    ```bash
    git clone https://github.com/khulnasoft/ai-presentation-generator.git
    cd ai-presentation-generator
    ```

2.  **Создайте файл `.env`:**

    Скопируйте файл `.env.example` в `.env`. Этот файл будет содержать ваши секретные ключи и настройки.

    ```bash
    cp .env.example .env
    ```

3.  **Настройте `.env`:**

    Откройте файл `.env` в текстовом редакторе и добавьте ваш API-ключ.

    ```ini
    # ═══════════════════════════════════════════════════════
    # AI Presentation Generator — Environment Variables
    # ═══════════════════════════════════════════════════════

    # --- LLM API Keys ---
    GEMINI_API_KEY=your-gemini-api-key-here
    # OPENAI_API_KEY=your-openai-api-key-here

    # --- JWT Auth ---
    # Сгенерируйте случайную строку для продакшена
    JWT_SECRET_KEY=change-me-to-a-random-secret-in-production

    # --- Ports ---
    FRONTEND_PORT=3000
    ```

    > **Важно**: `JWT_SECRET_KEY` должен быть заменён на длинную, случайную и надёжную строку в реальном продакшен-окружении.

4.  **Запустите приложение с помощью Docker Compose:**

    Эта команда соберёт Docker-образы для frontend и backend, создаст базу данных PostgreSQL и запустит все сервисы в фоновом режиме (`-d`).

    ```bash
    docker compose up -d --build
    ```

    Первый запуск может занять несколько минут, так как Docker будет скачивать базовые образы и устанавливать зависимости.

5.  **Откройте приложение в браузере:**

    После успешного запуска приложение будет доступно по адресу [http://localhost:3000](http://localhost:3000).

## Проверка статуса

Вы можете проверить статус запущенных контейнеров командой:

```bash
docker compose ps
```

Вы должны увидеть три работающих сервиса: `aipres-db`, `aipres-api`, `aipres-frontend`.

## Остановка приложения

Чтобы остановить все сервисы, выполните:

```bash
docker compose down
```

Для полного сброса, включая удаление базы данных, используйте флаг `-v`:

```bash
docker compose down -v
```
