# ADR-002: Переключение Instructor на JSON Mode для Gemini

**Дата:** 2026-02-24
**Статус:** Принято
**Авторы:** Manus AI

## Контекст

В ходе тестирования полного цикла генерации (коммит `fa9f7e2`) была обнаружена критическая ошибка: `S0_PlannerNode` не мог сгенерировать валидный план. LLM (Gemini 2.5 Flash через OpenAI-совместимый API) возвращал шаги плана как строки (`"default_api.S1_ContextAnalyzer(...)"`) вместо ожидаемых JSON-объектов. Это приводило к ошибке Pydantic-валидации в `ExecutionPlanSchema`.

Библиотека `instructor` по умолчанию использует `mode=Mode.TOOLS`, который опирается на `tool_calls` в ответе LLM. Gemini, в отличие от нативных моделей OpenAI, не всегда корректно формирует `tool_calls` через совместимый API, что и приводило к строковому представлению.

## Решение

Выбрано решение переключить `instructor` на `mode=Mode.JSON` во всех узлах (S0-S5). Этот режим заставляет LLM генерировать ответ в виде чистого JSON-объекта, который затем парсится `instructor`.

```python
# было
client = instructor.from_openai(
    OpenAI(model=model, ...),
    mode=instructor.Mode.TOOLS,
)

# стало
client = instructor.from_openai(
    OpenAI(model=model, ...),
    mode=instructor.Mode.JSON,
)
```

Дополнительно, в `ExecutionPlanSchema` был добавлен `field_validator` как fallback-механизм, который пытается распарсить строковые шаги с помощью `eval()`, если получает `str` вместо `PlanStep`. Это делает систему более устойчивой к подобным ошибкам в будущем.

## Последствия

**Положительные:**
- **Надёжность:** `Mode.JSON` работает стабильнее с Gemini через OpenAI-совместимый API.
- **Успешная генерация:** Проблема была полностью решена, полный цикл генерации заработал.
- **Устойчивость:** Fallback-валидатор в `ExecutionPlanSchema` обеспечивает дополнительный уровень защиты.

**Отрицательные:**
- **Меньшая гибкость LLM:** `Mode.TOOLS` в теории позволяет LLM выбирать несколько инструментов или текст + инструмент, в то время как `Mode.JSON` жёстко требует один JSON-объект. В нашем случае это не является проблемой, так как каждый узел ожидает один конкретный Pydantic-объект.
