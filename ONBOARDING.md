# Onboarding: Быстрый старт

**Версия:** 1.0
**Дата:** 2026-02-23

## 1. Цель

Этот документ поможет новому разработчику настроить окружение, запустить проект и сделать свой первый коммит **менее чем за 1 час**.

## 2. Шаг 1: Настройка окружения (15 минут)

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <URL репозитория>
    cd <имя репозитория>
    ```
2.  **Установите Poetry:**
    ```bash
    pip install poetry
    ```
3.  **Установите зависимости:**
    ```bash
    poetry install
    ```
4.  **Настройте pre-commit хуки:**
    ```bash
    poetry run pre-commit install
    ```
5.  **Создайте файл `.env`:**
    - Скопируйте `.env.example` в `.env`.
    - Заполните необходимые API-ключи (например, `GEMINI_API_KEY`).

## 3. Шаг 2: Запуск тестов (5 минут)

Убедитесь, что всё работает, запустив полный набор тестов:

```bash
poetry run pytest
```

Все тесты должны пройти успешно.

## 4. Шаг 3: Понимание архитектуры (30 минут)

Чтобы быстро понять, как устроен проект, прочитайте следующие документы в указанном порядке:

1.  **`README.md`** (2 минуты) — что это за проект?
2.  **`CONTEXT.md`** (2 минуты) — что происходит прямо сейчас?
3.  **`development_roadmap.md`** (5 минут) — что мы делаем сейчас и что дальше?
4.  **`engine_architecture_specification_v3_ru.md`** (15 минут) — как устроен движок? (Прочитайте разделы 1-5).
5.  **`technical_specification.md`** (10 минут) — что делают S1-S5, дизайн-система, база знаний.
6.  **Последний ADR** в `docs/adr/` (5 минут) — какое последнее важное решение мы приняли?

## 5. Шаг 4: Ваша первая задача (10 минут)

1.  Возьмите простую задачу из бэклога (например, с тегом `good first issue`).
2.  Создайте новую ветку: `feature/JIRA-XYZ-my-first-task`.
3.  Внесите небольшое изменение (например, добавьте комментарий или исправьте опечатку).
4.  Запустите тесты и линтеры:
    ```bash
    poetry run pytest
    poetry run ruff check .
    ```
5.  Сделайте коммит, следуя [Conventional Commits](https://www.conventionalcommits.org/):
    ```bash
    git commit -m "docs: add clarification to SharedStore docstring"
    ```
6.  Отправьте ветку на сервер и создайте Pull Request (PR).

Поздравляем, вы в команде!
